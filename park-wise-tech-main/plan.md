## ParkPilot — AI Parking Concierge

A chat-first parking assistant for Bengaluru. Users converse naturally to find, forecast, book, extend, and understand parking. The LLM drives everything via tools over a seeded mock dataset.

### Stack
- TanStack Start + Lovable Cloud (auth + Postgres)
- Vercel AI SDK via Lovable AI Gateway (`google/gemini-3-flash-preview`)
- AI Elements for chat UI
- Email/password + Google sign-in
- No user-profile fields beyond `auth.users` (roles not needed)

### Database (migrations)
- `parking_spots` — id, name, area, lat, lng, hourly_rate, capacity, current_occupancy, walk_minutes_landmark, notes (public read)
- `threads` — id, user_id, title, updated_at (RLS: owner)
- `messages` — id (uuid), thread_id, role, parts (jsonb), created_at (RLS: owner via thread)
- `bookings` — id, user_id, spot_id, start_at, end_at, price_inr, status (RLS: owner)
- Seed ~12 spots across Koramangala, Indiranagar, MG Road, HSR, Whitefield with varying rates, capacities, occupancy, walk distances

### Server
- `src/lib/ai-gateway.ts` — Lovable AI provider helper
- `src/routes/api/chat.ts` — POST handler:
  - `streamText` + `stopWhen: stepCountIs(50)`
  - Tools (Zod schemas):
    - `search_spots({ area, max_price, duration_hours, start_time })` — filters seeded spots, returns ranked list with availability + walk time
    - `forecast_demand({ spot_id, hours_ahead })` — synthesizes occupancy curve using base occupancy + heuristics for evening peak / rain / nearby events (mocked context signals)
    - `book_spot({ spot_id, start_at, end_at })` — needsApproval; inserts into `bookings`
    - `extend_booking({ booking_id, additional_hours })` — needsApproval; updates booking
    - `explain_price({ spot_id })` — returns one-sentence reason using mock context (events, time-of-day, weather)
  - Persists assistant message in `onFinish` (UUID generated by DB; AI SDK message id stored in text column for dedupe)
- `src/lib/threads.functions.ts` — `createThread`, `listThreads`, `loadMessages`, `deleteThread`, `renameThread` (auth-protected via `requireSupabaseAuth`)

### Frontend (AI Elements)
Install: `bun x ai-elements@latest add conversation message prompt-input shimmer tool`
- `src/routes/_authenticated.tsx` — gate via `supabase.auth.getUser()`
- `src/routes/_authenticated/index.tsx` — chat workspace:
  - Left: thread sidebar (list, new chat, rename/delete)
  - Right: `Conversation` + `Message`/`MessageResponse` + `PromptInput` (textarea + footer + submit)
  - Tool parts rendered via AI Elements `Tool`, with custom compact renderers:
    - `search_spots` → spot cards (name, price, walk, availability bar)
    - `forecast_demand` → mini sparkline of next 3h
    - `book_spot` / `extend_booking` → confirm card (approval UI)
    - `explain_price` → callout
  - Markdown via `MessageResponse`; assistant has no bubble; user bubble = `primary` / `primary-foreground`
  - Textarea auto-focus on mount/send/stream-end/thread-switch
  - Remount chat on thread change via `key={threadId}`
- `src/routes/login.tsx` — email/password + Google
- Custom logo: generated SVG/PNG mark (parking-pin + chat bubble), not Sparkles

### Design
- Warm Bengaluru-evening palette: deep indigo bg, saffron accent, soft cream foreground; oklch tokens in `src/styles.css`
- Inter font; rounded-2xl cards; subtle gradients on hero/empty state

### Technical details
- Use `requireSupabaseAuth` middleware on all thread/booking server fns
- DB-generated UUIDs for `messages.id`; store AI SDK `msg_...` in separate text column
- Check `{ error }` on every Supabase call
- `stopWhen: stepCountIs(50)`
- Forecast tool uses deterministic seeded function over (occupancy, hour-of-day, mock event flag, mock weather flag) — no real APIs
- Surface 429/402 gateway errors as toasts

### Out of scope (v1)
- Real maps, real payments, real sensor feeds
- Multi-user role admin
